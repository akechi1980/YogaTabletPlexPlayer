# 项目任务说明书

## 项目名称
Plex Poster Launcher

## 项目目标
开发一个运行在 Windows 上的轻量桌面工具，用于连接自有 Plex Media Server，显示媒体海报墙，支持基础搜索与筛选，点击资源后调用本机已安装的 VLC 外部播放器进行播放。

本项目不是完整 Plex 客户端，只做最小实用功能：
- 显示海报墙
- 浏览媒体列表
- 搜索资源
- 点击后外部调用 VLC 播放
- 尽量适配低配置旧设备

---

## 核心需求

### 1. 基础连接配置
程序首次启动时，提供一个简单配置界面或配置文件，至少支持以下参数：
- Plex 服务器地址，例如：`http://192.168.1.100:32400`
- Plex Token
- VLC 可执行文件路径，例如：`C:\Program Files\VideoLAN\VLC\vlc.exe`

要求：
- 配置可保存到本地
- 下次启动自动读取
- 可以在 UI 中修改并重新保存

---

### 2. 媒体库读取
连接 Plex 后，读取用户指定库中的内容并展示。

最低要求：
- 支持读取电影库
- 如实现成本不高，可额外支持剧集库
- 每个媒体项至少显示：
  - 标题
  - 年份（如果有）
  - 海报缩略图
  - ratingKey 或唯一标识

建议：
- 先只做电影库
- 剧集支持可以作为后续可选项

---

### 3. 海报墙展示
主界面是海报网格墙。

要求：
- 网格布局显示海报
- 支持滚动
- 支持懒加载 / 分页加载
- 点击某个海报可进入详情或直接播放
- 低配置机器上不要使用复杂动画
- UI 尽量朴素、轻量、稳定

详情信息最少显示：
- 标题
- 年份
- 简介（可选）
- 播放按钮

---

### 4. 搜索功能
需要有最基础的搜索功能。

要求：
- 顶部提供搜索框
- 支持按标题关键字过滤当前媒体列表
- 搜索可以是本地已加载数据过滤，不强制服务端实时搜索
- 输入后即时过滤，或者点击按钮过滤均可

可选增强：
- 按最近添加排序
- 按标题排序
- 按年份筛选

但这些都不是必须项，优先保证主流程可用

---

### 5. 外部 VLC 播放
点击播放后，不内嵌播放器，不使用 libVLC 嵌窗，直接调用外部 VLC。

要求：
- 使用系统进程启动 `vlc.exe`
- 将可播放地址作为参数传给 VLC
- 若本机未配置 VLC 路径，给出明确提示
- 程序本身不负责播放控制，只负责启动 VLC

注意：
- 优先采用最简单、最稳的方案
- 不追求 Plex 官方客户端那种播放状态同步、续播、转码控制等完整能力
- 只要在用户自己的固定 Plex 环境下可用即可

---

## 低配设备约束
目标机器可能是老旧 Windows 10 32 位设备，内存约 2GB。

因此必须优先考虑：
- 尽量轻量
- 少依赖大型运行时
- 少动画
- 缩略图不要太大
- 做本地图片缓存，避免重复加载海报
- 列表分页或分批加载，避免一次性加载全部海报
- 避免过度占用内存

如果某些框架对 32 位支持不佳，优先保证可运行与稳定，而不是界面华丽。

---

## 非目标（明确不做）
本项目第一阶段不要做以下功能：
- 不做 Plex 用户登录流程
- 不做多用户切换
- 不做转码策略选择
- 不做字幕切换
- 不做音轨切换
- 不做播放进度回传
- 不做继续观看同步
- 不做内嵌 VLC / libVLC
- 不做在线播放器控制条
- 不做下载功能
- 不做账户系统
- 不做复杂皮肤或主题系统

---

## 技术实现建议

### 推荐语言
- Rust

### HTTP / 数据处理建议
- 使用稳定成熟的 HTTP 客户端库
- 自己请求 Plex 接口
- 不强依赖不成熟的 Plex 专用 crate
- JSON / XML 哪种更顺手就用哪种，但实现需清晰可维护

### UI 建议
目标是“轻量和可运行优先”。

可接受思路：
1. Rust 轻量 GUI 方案
2. 若 GUI 框架选择困难，可先做一个最简单桌面窗口程序
3. 不要为了美观引入过重的 Web 技术栈，除非确认对低配机器影响可接受

### 配置存储
- 本地配置文件即可，例如 `config.toml` 或 `config.json`

### 图片缓存
- 将海报缓存到本地缓存目录
- 使用 Plex 海报地址下载缩略图后缓存
- 下次优先读本地缓存
- 可设置简单缓存命名规则，例如基于 ratingKey 命名

---

## Plex 侧功能拆分建议

### 最小必须实现
1. 读取配置
2. 测试 Plex 连通性
3. 获取媒体库列表
4. 选择目标库
5. 拉取媒体项列表
6. 获取海报地址
7. 下载并缓存海报
8. UI 显示海报墙
9. 搜索过滤
10. 点击播放后调用 VLC

### 可选增强
- 最近添加
- 记住上次选中的库
- 详情页显示简介
- 加一个“刷新媒体库”按钮

---

## 播放链路要求
Codex 在实现播放时，优先采用最稳妥方案。

可接受优先级：
1. 直接构造 Plex 可访问的播放 URL 并传给 VLC
2. 如果某些资源类型播放 URL 处理复杂，先只保证电影资源可播放
3. 允许先在固定局域网、自有服务器场景下可用，不要求兼容所有 Plex 部署形态

关键原则：
- 先可用
- 先支持最常见场景
- 不要过度设计

## 验收标准
以下条件都满足即可视为第一阶段完成：

1. 启动程序后可以填写并保存 Plex 地址、Token、VLC 路径
2. 可以成功连接 Plex 并读取至少一个电影库
3. 主界面能显示海报墙
4. 输入标题关键字可以过滤资源
5. 点击某个资源后，能成功启动外部 VLC 播放
6. 重启程序后配置不会丢失
7. 在较低配置 Windows 机器上仍保持基本可用

## 最终目标总结
这是一个“Plex 海报浏览 + 简单搜索 + 一键调用 VLC”的轻量启动器，而不是完整媒体中心。

请优先实现：
“能配置 → 能拉列表 → 能看海报 → 能搜索 → 能点开 VLC 播放”

只要这条主链路打通，就算第一阶段成功。
